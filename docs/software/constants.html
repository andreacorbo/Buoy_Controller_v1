Constants¶

The constants.py file contains the global system parameters.

CONFIG_DIR str Configuration files directory
CONFIG_TYPE str Configuration files type
LOG_DIR str Log files directory
LOG_LEVEL int Log level 0 = screen output, 1 = log to file
VERBOSE int Message verbosity 0 = nothing, 1 = shows device activity
DEVICE_STATUS dict Device status stored in the tools.utils.status_table {0:”OFF”, 1:”ON”, 2:”READY”}
DEVICES dict Device to uart number mapping {“device1_instance”:1, …}
UARTS dict Uart number to bus mapping {1:2, 2:4, 3:6, 4:1}
LEDS dict Board status to leds mapping red=1, green=2, yellow=3, blue=4 {“IO”:1, “PWR”:2, “RUN”:3, “SLEEP”:4}
TIMEOUT int Generic timeout in seconds.
WD_TIMEOUT int Watchdog timer timeout in milliseconds.
ESC_CHAR str Escape character that has to be typed 3 times to break main loop.
PASSWD str Login password.
LOGIN_ATTEMPTS int Maximum number of login attempts.
SESSION_TIMEOUT int Session timeout in seconds.
MEDIA list Available data files locations [“/sd”, “/flash”]
DATA_DIR str Data files directory.
DATA_FILE_NAME str Data files conventional name.
DATA_SEPARATOR str Record data separator.
DATA_ACQUISITION_INTERVAL int Seconds between two data acquisitions, valid unless specified in the SCHEDULER parameter.
TASK_SCHEDULER dict Bidimensional dictionary used to schedule any kind of task. Each device is identified by it’s name and instance number. The task name corresponds to the object method to invoke. The interval is in seconds. {“device1_instance1”:{“task1”:interval, …}, …}
TMP_FILE_PFX str Partially sent data file prefix.
SENT_FILE_PFX str Totally sent data file prefix.
BUF_DAYS int Days data files will be kept in the transmission queue. Elder files will be ignored.

Warning

Due to limited flash capacity, redirect data to this location in case of an SD failure it’s not possible.

Note

To set the data acquisition interval for a certain device that is different from the value specified by the DATA_ACQUISITION_INTERVAL parameter, insert a task “log” for that device.
